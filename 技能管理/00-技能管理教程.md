## 一、Skill 系统详解

### 1.1 什么是 Skill？

Skill 是 OpenClaw 的能力扩展包，告诉 Agent"怎么做特定任务"。

**比喻**：像员工的专业培训手册。

### 1.2 Skill 的完整结构

```
skills/my-skill/
├── SKILL.md           ← 技能描述（必须）
├── references/        ← 参考资料（可选）
│   └── api-doc.md
├── scripts/           ← 执行脚本（可选）
│   └── run.py
└── assets/            ← 资源文件（可选）
    └── template.json
```

### 1.3 SKILL.md 标准格式

```markdown
---
name: my-skill
description: 技能描述
version: 1.0.0
---

# 技能名称

## 触发条件
- 用户提到关键词"XXX"
- 检测到特定文件类型
- 特定命令触发

## 执行步骤
1. 第一步：读取输入
2. 第二步：处理数据
3. 第三步：输出结果

## 注意事项
- 需要 Python 环境
- 依赖 XX 库
- 错误处理方式

## 示例
用户："帮我做 XXX"
Agent：加载 my-skill 技能，执行工作流
```

### 1.4 Skill 的分类

| 类型 | 位置 | 作用域 | 示例 |
|------|------|--------|------|
| **用户级** | `~/.openclaw/skills/user-level/` | 全局可用 | 网页抓取、数据分析 |
| **项目级** | `{workspace}/.openclaw/skills/project-level/` | 仅当前项目 | 项目特定工作流 |

## 二、从 ClawHub 安装 Skill

### 2.1 ClawHub 技能市场

- **中文网**：https://clawhubcn.com/
- **官方网站**：https://clawhub.ai/

### 2.2 搜索和安装

```bash
# 搜索技能
openclaw skills search <关键词>

# 查看技能详情
openclaw skills show <skill-name>

# 安装技能
openclaw skills install <skill-name>

# 查看已安装技能
openclaw skills list

# 更新技能
openclaw skills update <skill-name>

# 卸载技能
openclaw skills remove <skill-name>
```

### 2.3 实战：安装社区技能

```bash
# 搜索代码审查相关技能
openclaw skills search "code review"

# 安装选中的技能
openclaw skills install code-review-pro

# 验证安装
openclaw skills list | grep code-review
```

## 三、创建自定义 Skill

### 3.1 实战 1：代码审查 Skill

#### 步骤 1：创建目录

```bash
mkdir -p ~/.openclaw/workspace/skills/code-review
cd ~/.openclaw/workspace/skills/code-review
```

#### 步骤 2：编写 SKILL.md

```markdown
---
name: code-review
description: 代码质量审查技能
version: 1.0.0
---

# Code Review

## 触发条件
- 用户提到"审查代码"、"review"、"代码检查"
- 用户发送代码片段并要求审查

## 审查维度
1. 🔴 **安全**: SQL注入、XSS、硬编码密钥、未授权访问
2. 🟡 **性能**: N+1查询、大对象传递、未使用变量
3. 🟢 **规范**: 命名一致性、注释完整度、错误处理

## 输出格式
## 🔴 严重
- 第X行: 具体问题描述

## 🟡 警告
- 第X行: 具体问题描述

## 🟢 建议
- 第X行: 具体优化建议

## 注意事项
- 只审查用户提供的代码
- 不主动重构
- 给出具体修改代码示例
```

#### 步骤 3：测试

```
> 审查这段代码：[粘贴代码]
```

## 四、Skill 调试与测试

### 4.1 日志追踪

```bash
# 查看 Skill 执行日志
tail -f ~/.openclaw/gateway/gateway.log | grep -i skill

# 查看特定 Skill 日志
openclaw skills logs <skill-name>
```

### 4.2 断点测试

在 SKILL.md 中添加调试输出：

```markdown
## 调试模式
- 执行前输出：即将执行 XXX 步骤
- 执行后输出：XXX 步骤完成，结果：YYY
```

### 4.3 错误定位

| 错误类型 | 排查方法 | 解决方案 |
|---------|---------|---------|
| 触发词不匹配 | 检查 SKILL.md 触发条件 | 补充更多触发词 |
| 脚本执行失败 | 查看 scripts/ 目录权限 | `chmod +x scripts/*.py` |
| 依赖缺失 | 检查 references/ 文件 | 安装缺失的依赖 |
| 输出格式错误 | 对比预期输出 | 修正 SKILL.md 格式 |

---

### 4.4 Skill 测试策略

#### 单元测试

测试单个功能点：

```bash
# 创建测试目录
mkdir -p skills/code-review/tests

# 编写测试用例
cat > skills/code-review/tests/test_security.py << 'EOF'
def test_sql_injection_detection():
    code = "SELECT * FROM users WHERE id = " + user_input
    assert detect_sql_injection(code) == True

def test_xss_detection():
    code = "<script>alert('xss')</script>"
    assert detect_xss(code) == True
EOF
```

#### 集成测试

测试完整工作流：

```bash
# 创建集成测试脚本
cat > skills/code-review/tests/integration_test.sh << 'EOF'
#!/bin/bash
echo "测试代码审查完整流程..."
openclaw skills test code-review --input "test_code.py"
echo "测试完成"
EOF

chmod +x skills/code-review/tests/integration_test.sh
```

#### 回归测试

确保修改不破坏现有功能：

```bash
# 创建回归测试清单
cat > skills/code-review/tests/regression.md << 'EOF'
# 回归测试清单

## 测试用例
1. 简单函数审查 → 应通过
2. SQL 注入检测 → 应标记严重
3. XSS 漏洞检测 → 应标记严重
4. 性能问题检测 → 应标记警告
5. 空代码输入 → 应返回错误提示
EOF
```

## 五、技能生命周期

```
创建 → 安装 → 触发 → 执行 → 优化 → 共享
```

1. **创建**: 编写 SKILL.md
2. **安装**: 放入 skills/ 目录
3. **触发**: 用户请求匹配触发条件
4. **执行**: 按照 SKILL.md 执行
5. **优化**: 根据执行结果改进
6. **共享**: 推送到 ClawHub

## 🔍 常见问题排查（Q&A）

### Q1: Skill 不被触发？

**A:** 检查以下几点：
1. 确认技能已安装：`openclaw skills list`
2. 检查 SKILL.md 的触发词配置
3. 查看技能日志：`openclaw skills logs <name>`
4. 手动测试：`openclaw skills test <name>`

### Q2: 技能执行后报错？

**A:** 
1. 检查 SKILL.md 中的工具引用是否正确
2. 确认工具已在 `TOOLS.md` 中配置
3. 查看 Gateway 日志：`tail -f ~/.openclaw/gateway/gateway.log`

### Q3: 如何共享技能给团队？

**A:**
1. 将技能目录打包：`tar -czf my-skill.tar.gz skills/my-skill/`
2. 推送到 ClawHub：`openclaw skills publish my-skill`
3. 或分享到团队共享目录

### Q4: 技能更新后不生效？

**A:** 刷新技能缓存：
```bash
openclaw skills refresh
```

### Q5: 如何测试技能的触发词？

**A:**
```bash
# 使用测试模式
openclaw skills test <name> --trigger "测试触发词"

# 查看匹配结果
# 应显示：技能已加载，准备执行
```
