# 📘 第五阶段：OpenClaw 故障排查手册（1 天）

> **培训阶段**：第五阶段 - 生产级运维
> **目标**：遇到报错不要慌，系统化排查和解决问题
> **预计用时**：3-4 小时

---

## 📋 学习目标清单

完成本阶段培训后，学员将能够：

1. **系统化排查故障**：按照标准流程诊断和解决各类问题
2. **解决安装问题**：处理 npm 安装、权限、Node.js 版本等常见问题
3. **解决 Gateway 问题**：处理端口占用、配置错误、启动失败等问题
4. **解决模型问题**：处理 API Key 无效、403/401 错误、速率限制等问题
5. **解决 Skill 问题**：处理技能不触发、执行失败、输出错误等问题
6. **解决 Cron 问题**：处理任务不触发、执行无输出、配置错误等问题
7. **解决平台问题**：处理企业微信、飞书、钉钉等平台接入问题

## 一、快速诊断流程

```
OpenClaw 出问题了？
  ↓
1. 看日志 → openclaw logs 或 tail -f ~/.openclaw/gateway/gateway.log
  ↓
2. 检查 Gateway 状态 → openclaw gateway status
  ↓
3. 检查模型连接 → curl 测试 API
  ↓
4. 检查 Skill/Cron 配置 → openclaw skills list / openclaw cron list
  ↓
5. 搜本章 → 有没有匹配的报错？
  ↓
6. 按方案解决 → 验证问题是否修复
```

## 二、安装失败

### 报错：`command not found: openclaw`

**原因**：未安装或不在 PATH 中

**解决**：
```bash
# 检查是否安装
npm list -g openclaw

# 如果没有，安装
npm install -g openclaw

# 或使用 pnpm
pnpm add -g openclaw

# 验证 PATH
echo $PATH
```

---

### 报错：`npm ERR! code EACCES`

**原因**：权限不足

**解决**：
```bash
# 方法 A：使用淘宝镜像 + 修复权限
npm config set registry https://registry.npmmirror.com
mkdir -p ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.zshrc
source ~/.zshrc
npm install -g openclaw
```

---

### 报错：`Node.js version too old`

**原因**：需要 Node 22+（推荐 24）

**解决**：
```bash
# 检查当前版本
node --version

# 使用 nvm 升级
nvm install 24
nvm use 24
nvm alias default 24
```

## 三、Gateway 启动失败

### 报错：`Gateway failed to start`

**原因**：端口被占用或配置错误

**解决**：
```bash
# 检查端口占用
lsof -i :18789

# 杀掉占用进程
kill -9 <PID>

# 或使用其他端口
openclaw gateway start --port 18790
```

---

### 报错：`config.json parse error`

**原因**：配置文件 JSON 格式错误

**解决**：
```bash
# 检查配置文件
cat ~/.openclaw/openclaw.json | jq .

# 如果 jq 报错，修复 JSON 格式
# 注意逗号、引号、括号
```

---

### 报错：`model provider not found`

**原因**：models.json 配置缺失

**解决**：
```bash
# 检查模型配置
cat ~/.openclaw/agents/main/agent/models.json

# 确保 providers 下有有效配置
```

## 四、模型问题

### 报错：`403 Forbidden`

**原因**：API Key 无效或模型未开通

**解决**：
1. 检查 models.json 中的 apiKey 是否正确
2. 登录模型提供商控制台确认模型已开通
3. 确认 baseUrl 是否正确

**验证配置**：
```bash
curl -X POST "https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer sk-xxx" \
  -d '{"model": "qwen-plus", "messages": [{"role": "user", "content": "hi"}]}'
```

---

### 报错：`401 Unauthorized`

**原因**：API Key 无效或过期

**解决**：
```json
// 检查 models.json
{
  "providers": {
    "my-provider": {
      "apiKey": "sk-xxx"  // 确认 Key 正确
    }
  }
}
```

---

### 报错：`Rate limit exceeded`

**原因**：QPS 超限

**解决**：
- 降低请求频率
- 切换到备用模型
- 联系模型提供商提高限额

---

### 问题：回复很慢（>30 秒）

**排查步骤**：
1. 检查模型端点响应时间
2. 检查网络延迟
3. 换用更快的模型

---

## 五、Skill 问题

### 问题：Skill 不被触发

**原因**：Skill 未正确安装或触发条件不匹配

**解决**：
```bash
# 检查已安装技能
openclaw skills list

# 检查 Skill 目录
ls ~/.openclaw/workspace/skills/

# 检查 SKILL.md 格式
cat ~/.openclaw/workspace/skills/<name>/SKILL.md
```

确保 SKILL.md 包含正确的触发条件。

---

### 问题：`openclaw skills install` 失败

**原因**：ClawHub 连接失败或技能不存在

**解决**：
```bash
# 检查网络连接
curl -I https://clawhub.ai

# 手动安装
mkdir -p ~/.openclaw/workspace/skills/<name>
# 手动创建 SKILL.md
```

---

### 问题：Skill 执行后报错

**原因**：Skill 定义的工具未正确配置

**解决**：
1. 检查 SKILL.md 中的工具引用是否正确
2. 确认工具已在 `~/.openclaw/workspace/TOOLS.md` 中配置
3. 查看日志：`openclaw logs`

---

## 六、Cron 任务问题

### 问题：Cron 任务不触发

**原因**：schedule 配置错误或 sessionTarget 不匹配

**排查**：
```bash
# 列出所有 Cron 任务
openclaw cron list

# 检查任务配置
openclaw cron detail <job-id>
```

**常见错误**：

| 错误配置 | 正确配置 |
|---------|---------|
| `sessionTarget: "main"` + `payload.kind: "agentTurn"` | `"main"` 只能配 `"systemEvent"` |
| `sessionTarget: "isolated"` + `payload.kind: "systemEvent"` | `"isolated"` 只能配 `"agentTurn"` |
| 时区未指定 | 添加 `"tz": "Asia/Shanghai"` |

---

### 问题：Cron 任务执行但没输出

**原因**：delivery 模式未配置或 webhook 地址错误

**解决**：
```json
// 确保 delivery 配置正确
{
  "delivery": {
    "mode": "announce"  // 或 "webhook"
  }
}
```

---

## 七、消息平台问题

### 企业微信不回复

**原因**：Webhook 只能发送，不能接收

**解决**：需要使用企业微信自建应用

---

### Telegram 连接超时

**解决**：
```bash
# 配置代理
export HTTPS_PROXY=http://127.0.0.1:7890
```

---

### 飞书事件回调 404

**解决**：
1. 确认 Gateway 正常运行
2. 检查回调地址格式：`http://你的 IP:端口/webhook/feishu`
3. 确认端口已开放

---

## 八、性能问题

### 问题：上下文溢出

**原因**：上下文窗口设置太小或消息过多

**解决**：
```json
// 在 models.json 中调整
{
  "contextWindow": 128000,  // 增大上下文
  "maxTokens": 8192
}
```

---

### 问题：内存占用高

**解决**：
- 启用自动压缩（Compaction）
- 减少同时活跃的 Session 数量
- 定期清理旧 Session

---

## 九、权限问题

### 报错：`exec permission denied`

**原因**：exec 安全模式限制

**解决**：
```json
// 在 openclaw.json 中调整
{
  "exec": {
    "mode": "allowlist"  // 或 "full"（谨慎使用）
  }
}
```

---

### 报错：`file access denied`

**原因**：文件沙箱限制

**解决**：
```json
// 在 openclaw.json 中配置允许路径
{
  "sandbox": {
    "allowedPaths": ["/Users/zhushuiqing/.openclaw/workspace"]
  }
}
```
