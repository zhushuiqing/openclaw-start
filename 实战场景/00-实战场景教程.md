## 二、实战场景

### 场景 1：舆情监控自动化

#### 业务背景
公司需要实时监测品牌和行业舆情，人工巡查效率低、容易漏。

#### 目标
用 OpenClaw 搭建自动化舆情监控，每 2 小时自动抓取、分析、推送企微群。

#### 实施步骤

**步骤 1：创建 Cron 定时任务**

```json
{
  "name": "public-opinion-monitor",
  "schedule": {
    "kind": "cron",
    "expr": "0 */2 * * *",
    "tz": "Asia/Shanghai"
  },
  "payload": {
    "kind": "agentTurn",
    "message": "执行舆情监控任务..."
  },
  "sessionTarget": "isolated",
  "delivery": {
    "mode": "announce",
    "channel": "wecom"
  }
}
```

**步骤 2：配置监控任务**

```
执行舆情监控任务：
1. 搜索最近 2 小时内关于"[公司名]"的新闻和社交媒体讨论
2. 搜索行业关键词：[关键词1, 关键词2]
3. 分析情绪倾向（正面/负面/中性）
4. 如果有重大负面舆情，立即标记并推送
5. 汇总结果：
   - 新增讨论数
   - 情绪分布
   - 关键内容摘录（前 5 条）
   - 预警信息（如有）
```

#### 验证清单
- [ ] Cron 任务按时触发
- [ ] 舆情数据能正常抓取
- [ ] 企微群能收到推送消息
- [ ] 负面舆情能及时预警

### 场景 2：自动化竞品分析

#### 业务背景
竞品动态需要人工跟踪，耗时且容易遗漏关键信息。

#### 目标
每天自动收集竞品动态，生成分析报告，推送给团队。

#### 实施步骤

**步骤 1：使用多 Agent 协作**

```
Agent 1（Collector）：搜索竞品A、竞品B的最新新闻和公告
Agent 2（Tracker）：检查竞品A、竞品B的产品更新日志
Agent 3（Monitor）：搜索竞品在 Twitter、小红书上的讨论热度
```

**步骤 2：汇总分析**

```
- 合并三个 Agent 的结果
- 识别关键趋势
- 生成竞品动态报告
```

**步骤 3：定时执行**

```bash
cron add --name "竞品日报" --schedule "0 9 * * *"
```

#### 输出模板

```
📊 竞品日报 | 2026-04-24

竞品A：
- 发布新版本 3.2，新增 AI 功能
- 获得 B 轮融资 5000 万美元

竞品B：
- 价格下调 15%
- 用户投诉性能问题增加

🔍 建议关注：
- 竞品A的 AI 功能对我们影响较大
```

### 场景 3：智能客户关系管理

#### 业务背景
客户跟进靠人工记忆，容易遗忘重要跟进节点。

#### 目标
自动提醒客户跟进、记录交互、生成跟进建议。

#### 实施步骤

**步骤 1：录入客户信息**

```
记录客户张三，公司ABC，上次联系4月20日，需求：采购方案
```

**步骤 2：设置跟进提醒**

```bash
cron add --name "跟进张三" --at "2026-04-28T10:00:00+08:00"
payload: "提醒：联系客户张三，了解采购方案反馈"
```

**步骤 3：自动跟进建议**

```
查看客户张三的交互记录，分析：
- 上次沟通内容
- 客户关心的问题
- 建议的下一步动作
```

### 场景 4：营销内容自动化

#### 业务背景
多平台内容分发（公众号、小红书、企微）耗时耗力。

#### 目标
一次生成，多平台自动适配并分发。

#### 实施步骤

**步骤 1：生成核心内容**

```
为主题'[营销主题]'生成营销内容：
- 核心文案（200 字）
- 3 个关键卖点
- 目标受众画像
- 行动号召
```

**步骤 2：多平台适配**

```
Agent 1：将核心内容扩展为公众号文章，1500 字
Agent 2：将核心内容改写为小红书风格，500 字
Agent 3：将核心内容改写为企微群消息，100 字
```

**步骤 3：定时推送**

```bash
cron add --name "营销推送" --schedule "0 10 * * 3,5"
```

### 场景 5：每日晨报自动生成

#### 业务背景
每天花 30 分钟浏览各种信息源，效率低且容易遗漏重要内容。

#### 目标
每天早 8 点自动生成晨报，推送企微群。

#### 实施步骤

```
生成每日晨报：
1. 行业新闻（前 5 条）
2. 竞品动态
3. 政策/监管更新
4. 市场数据（股市、汇率等关键指标）
5. 今日待办提醒
6. 日历提醒（今天有哪些会议）
```

**定时执行：**

```bash
cron add --name "每日晨报" --schedule "0 8 * * 1-5"
```

### 场景 6：会议自动化

#### 业务背景
会议后需要整理纪要、分配待办、跟进执行，流程繁琐。

#### 目标
会议结束自动生成纪要，分配待办，设置跟进提醒。

#### 实施步骤

**步骤 1：自动生成纪要**

```
根据以下会议记录，生成会议纪要：
- 会议主题、时间、参会人
- 讨论要点（分条列出）
- 决策结论
- 待办事项（责任人 + 截止日期）
```

**步骤 2：创建待办提醒**

```bash
cron add --name "待办-张三-提交方案" --at "2026-04-28T18:00:00+08:00"
```

## 🔍 常见问题排查

### Q1: Cron 任务不触发？
**A:** 检查 Cron 表达式和时区配置，确认 sessionTarget 与 payload.kind 匹配。

### Q2: 多 Agent 协作失败？
**A:** 检查各 Agent 配置是否正确，确认通信机制（文件/消息）正常工作。

### Q3: 消息推送失败？
**A:** 确认平台接入正常，检查 delivery 配置和 webhook URL。

### Q4: 数据抓取不完整？
**A:** 优化搜索关键词，增加搜索范围，检查网络连通性。

### Q5: 内容生成质量差？
**A:** 优化提示词，提供更多上下文信息，使用更强大的模型。

### Q6: 如何优化性能？
**A:** 减少不必要的 Agent 数量，合并 Cron 任务，使用缓存。

### Q7: 如何备份配置？
**A:** 备份 `~/.openclaw/` 目录，妥善保管平台凭证和 API Key。
