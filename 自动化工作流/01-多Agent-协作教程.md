## 一、多 Agent 协作核心概念

### 1.1 什么是多 Agent 协作？

一个 Agent 能力有限。多个 Agent 分工合作，效率更高。

**比喻**：像一个团队，每个人负责自己擅长的部分。

### 1.2 多 Agent vs 单 Agent 对比

| 场景 | 单 Agent | 多 Agent | 优势 |
|------|---------|---------|------|
| 代码开发 | 写 + 审查 | 专职写 + 专职审 | 质量更高 |
| 新闻系统 | 抓取 + 摘要 + 发送 | 各自独立并行 | 速度更快 |
| 内容创作 | 一人全包 | 研究 + 写作 + 审校 | 专业度更高 |
| 客户服务 | 通用回答 | 分类 + 回答 + 跟进 | 响应更精准 |

## 二、Agent 角色与模型选择

### 2.1 Agent 分工表

| Agent | 角色 | 适合模型 | 任务 | 特点 |
|-------|------|---------|------|------|
| Researcher | 研究员 | qwen-turbo | 信息收集 | 速度快，成本低 |
| Writer | 作者 | qwen-plus | 内容生成 | 平衡速度与质量 |
| Reviewer | 审查员 | qwen-max | 质量把关 | 高质量，深度分析 |
| Manager | 管理者 | qwen-max | 协调调度 | 复杂决策能力 |
| Sender | 发送者 | qwen-turbo | 消息推送 | 简单任务，低成本 |

### 2.2 模型选择策略

```
任务复杂度
    │
    ├── 简单任务（数据收集、格式转换）→ qwen-turbo
    │
    ├── 中等任务（内容生成、代码编写）→ qwen-plus
    │
    └── 复杂任务（代码审查、决策分析）→ qwen-max
```

## 三、实战案例

### 实战 1：双 Agent 代码审查

#### 场景

Agent A 写代码 → Agent B 审查 → Agent A 修复

#### 配置

```json
{
  "agents": {
    "writer": {
      "model": "qwen-plus",
      "role": "负责编写代码",
      "workspace": "~/.openclaw/agents/writer"
    },
    "reviewer": {
      "model": "qwen-max",
      "role": "负责审查代码质量",
      "workspace": "~/.openclaw/agents/reviewer"
    }
  }
}
```

#### 工作流程

```
用户提交需求
  ↓
Agent A（Writer）实现代码
  ↓
Agent B（Reviewer）审查
  ↓
如果通过 → 合并
如果不通过 → Agent A 修复
  ↓
循环直到通过
```

#### 实施步骤

```bash
# 1. 为每个 Agent 创建独立配置
mkdir -p ~/.openclaw/agents/writer
mkdir -p ~/.openclaw/agents/reviewer

# 2. 配置 writer 的 SKILL.md
cat > ~/.openclaw/agents/writer/SKILL.md << 'EOF'
# 代码编写 Agent

## 职责
- 根据需求编写代码
- 遵循代码规范
- 添加必要的注释

## 输出
- 代码文件保存到指定目录
- 生成代码说明文档
EOF

# 3. 配置 reviewer 的 SKILL.md
cat > ~/.openclaw/agents/reviewer/SKILL.md << 'EOF'
# 代码审查 Agent

## 职责
- 审查代码质量
- 检查潜在 bug
- 提出改进建议

## 审查标准
- 代码可读性
- 性能优化
- 安全性
- 测试覆盖率
EOF
```

### 实战 2：三 Agent 新闻系统

#### 场景

新闻抓取 → AI 摘要 → 推送发送

#### 配置

```
Agent A（Collector）: 抓取新闻
  ↓
Agent B（Summarizer）: 生成摘要
  ↓
Agent C（Sender）: 发送到企业微信
```

#### 使用 Cron 实现

```json
[
  {
    "name": "news-collect",
    "schedule": { "kind": "cron", "expr": "0 7 * * *" },
    "payload": { "kind": "agentTurn", "message": "抓取今日新闻" },
    "sessionTarget": "isolated"
  },
  {
    "name": "news-summarize",
    "schedule": { "kind": "cron", "expr": "5 7 * * *" },
    "payload": { "kind": "agentTurn", "message": "生成新闻摘要" },
    "sessionTarget": "isolated"
  },
  {
    "name": "news-send",
    "schedule": { "kind": "cron", "expr": "10 7 * * *" },
    "payload": { "kind": "agentTurn", "message": "发送摘要到企业微信" },
    "sessionTarget": "isolated"
  }
]
```

#### 实施步骤

```bash
# 1. 创建三个 Agent 的工作目录
mkdir -p ~/.openclaw/agents/{collector,summarizer,sender}

# 2. 配置每个 Agent 的 SKILL.md
# collector: 负责新闻抓取
# summarizer: 负责摘要生成
# sender: 负责消息推送

# 3. 配置 Cron 任务，设置时间间隔
# 注意：每个任务之间留出足够的时间间隔
```

## 四、Agent 间通信机制

### 4.1 文件传递方式

```
Agent A 输出 → 保存到文件 → Agent B 读取输入
```

**示例：**

```bash
# Agent A 保存结果
echo "调研结果..." > /tmp/research-output.md

# Agent B 读取并处理
cat /tmp/research-output.md | process
```

### 4.2 共享状态方式

使用共享的 MEMORY.md 或状态文件：

```bash
# 更新共享状态
echo "任务 A 完成" >> ~/.openclaw/workspace/SHARED_STATE.md

# 其他 Agent 读取状态
cat ~/.openclaw/workspace/SHARED_STATE.md
```

### 4.3 消息队列方式（高级）

对于复杂场景，可以使用消息队列：

```bash
# 使用 Redis 或其他消息队列
# Agent A 发送消息
redis-cli LPUSH task-queue "task-data"

# Agent B 接收消息
redis-cli RPOP task-queue
```

## 🔍 常见问题排查

### Q1: Agent 之间无法通信？

**A:** 检查以下几点：
1. 确认共享目录权限正确：`ls -la /tmp/agent-shared/`
2. 检查文件是否正确写入：`cat /tmp/agent-shared/output.md`
3. 确认接收方 Agent 正在运行：`openclaw agent list`
4. 查看日志确认是否有错误：`openclaw logs`

---

### Q2: 某个 Agent 执行特别慢？

**A:** 可能原因：
1. 模型选择不当（简单任务用了大模型）
2. 上下文过长导致处理缓慢
3. 网络延迟（云端模型）
4. 优化建议：换用更快的模型或减少上下文

---

### Q3: 多 Agent 系统资源占用过高？

**A:** 优化建议：
1. 减少同时运行的 Agent 数量
2. 使用更小的模型处理简单任务
3. 增加 Agent 间的时间间隔
4. 定期清理临时文件和日志

---

### Q4: 如何调试多 Agent 系统？

**A:** 使用以下命令：
```bash
# 查看所有 Agent 状态
openclaw agent list

# 查看特定 Agent 日志
openclaw logs --agent writer

# 查看 Cron 任务状态
openclaw cron list

# 手动触发某个 Agent 任务
openclaw agent run writer --input "test data"
```

---

### Q5: Agent 执行结果不一致？

**A:** 可能原因：
1. 模型温度设置过高（随机性大）
2. 上下文不完整
3. 提示词不够明确
4. 解决：降低 temperature，明确提示词，确保上下文完整

---

### Q6: 如何备份多 Agent 配置？

**A:** 备份以下位置：
- Agent 配置：`~/.openclaw/agents/`
- 团队配置：`~/.openclaw/teams/`
- 技能文件：`~/.openclaw/workspace/skills/`
- 共享状态：`~/.openclaw/workspace/SHARED_STATE.md`

---

### Q7: 如何扩展更多 Agent？

**A:** 步骤：
1. 创建新的 Agent 配置目录
2. 编写 SKILL.md 定义职责
3. 在团队配置中添加新成员
4. 更新通信机制和流水线
5. 测试新 Agent 能否正常工作
