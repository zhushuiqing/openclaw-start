## 一、自动化工作流核心概念

### 1.1 什么是自动化工作流？

自动化工作流让 OpenClaw 在**无人值守**时自动执行任务，例如：
- 每天早上发送新闻简报
- 定期检查邮件并回复
- 监控服务器状态并告警
- 每周自动生成项目进度报告

### 1.2 三大核心机制对比

| 机制 | 适用场景 | 时间精度 | 上下文 | 配置复杂度 |
|------|---------|---------|--------|-----------|
| **Cron** | 精确定时任务 | 分钟级 | 独立 Session | 中等 |
| **Heartbeat** | 周期性检查 | ~30 分钟 | 主 Session | 简单 |
| **Follow-up** | 链式任务 | 立即 | 当前 Session | 简单 |

## 二、核心机制详解

### 2.1 Cron 定时任务

OpenClaw 内置的 Cron 系统，支持精确时间调度。

**基本语法：**

```
cron add → schedule + payload → 自动执行
```

**配置示例：**

```json
{
  "schedule": {
    "kind": "cron",
    "expr": "0 9 * * *",
    "tz": "Asia/Shanghai"
  },
  "payload": {
    "kind": "agentTurn",
    "message": "生成今天的新闻简报并发送到企业微信"
  },
  "sessionTarget": "isolated"
}
```

**常用 Cron 表达式：**

| 表达式 | 含义 | 示例场景 |
|--------|------|---------|
| `0 9 * * *` | 每天 9:00 | 每日晨报 |
| `*/30 * * * *` | 每 30 分钟 | 邮件检查 |
| `0 * * * *` | 每小时整点 | 服务器监控 |
| `0 17 * * 5` | 每周五 17:00 | 周报生成 |
| `0 8 * * 1-5` | 工作日 8:00 | 工作日晨报 |

### 2.2 Follow-up 跟进机制

Agent 完成当前任务后，自动执行后续任务。

**链式执行流程：**

```
任务 A 完成 → 自动触发任务 B → 完成后触发任务 C
     ↓              ↓                  ↓
  收集数据      生成报告          发送通知
```

**配置方式：**

在 SKILL.md 中定义 follow-up 规则：

```markdown
## 完成后
- 如果发送成功 → 记录到日志
- 如果发送失败 → 10 分钟后重试
```

### 2.3 Heartbeat 心跳检查

周期性唤醒 Agent，执行检查任务。

**HEARTBEAT.md 配置示例：**

```markdown
# HEARTBEAT.md

## 检查清单（每 30 分钟）
1. 检查未读邮件
2. 检查日历（2 小时内有会议？）
3. 检查天气（外出需要带伞？）

## 发送消息条件
- 有重要邮件
- 会议即将开始
- 天气异常
```

## 三、实战案例

### 实战 1：创建每日新闻简报

#### 场景

每天早上 9:00 自动生成新闻摘要，发送到企业微信。

#### 步骤 1：创建技能

```bash
mkdir -p ~/.openclaw/workspace/skills/news-brief
```

**SKILL.md:**

```markdown
# 新闻简报

## 触发条件
- 定时触发（每天 9:00）

## 执行步骤
1. 使用 web_search 获取今日新闻
2. 按类别整理（军事/经济/科技）
3. 生成 HTML 格式简报
4. 发送到企业微信
```

#### 步骤 2：配置 Cron

```json
{
  "name": "daily-news-brief",
  "schedule": {
    "kind": "cron",
    "expr": "0 9 * * *",
    "tz": "Asia/Shanghai"
  },
  "payload": {
    "kind": "agentTurn",
    "message": "生成今天的新闻简报并发送到企业微信"
  },
  "sessionTarget": "isolated",
  "delivery": {
    "mode": "announce",
    "channel": "wecom"
  }
}
```

### 实战 3：服务器监控

#### 场景

每小时检查服务器状态，异常时发送告警。

#### 创建技能

```bash
mkdir -p ~/.openclaw/workspace/skills/server-monitor
```

**SKILL.md:**

```markdown
# 服务器监控

## 触发条件
- 定时触发（每小时）

## 执行步骤
1. 使用 exec 工具执行监控命令：
   - `df -h` 检查磁盘空间
   - `free -m` 检查内存使用
   - `top -l 1` 检查 CPU 使用
2. 判断阈值：
   - 磁盘 > 80% → 告警
   - 内存 > 90% → 告警
   - CPU > 80% → 告警
3. 发送告警消息

## 告警模板
🚨 服务器告警
{{metric}}: {{current_value}} (阈值: {{threshold}})
时间: {{timestamp}}
```

## 四、高级工作流模式

### 4.1 组合模式：Cron + Follow-up

```
Cron 触发（每天 9:00）
  ↓
任务 A：收集今日新闻
  ↓ (Follow-up)
任务 B：生成简报
  ↓ (Follow-up)
任务 C：发送到企业微信
  ↓ (Follow-up)
任务 D：记录执行日志
```

### 4.2 组合模式：Heartbeat + Cron

```
Heartbeat（每 30 分钟检查）
  ↓ 发现异常
Cron 触发告警任务
  ↓
发送告警通知
```

### 4.3 错误处理与重试机制

```markdown
## 错误处理策略
- 网络错误 → 等待 5 分钟后重试，最多 3 次
- 数据错误 → 记录错误日志，跳过本次执行
- 权限错误 → 立即告警管理员
```

## 🔍 常见问题排查

### Q1: Cron 任务到了时间没有触发？

**A:** 按以下步骤排查：
1. 检查 Cron 表达式是否正确：`openclaw cron list`
2. 确认时区配置：确保 `tz` 字段设置为 `Asia/Shanghai`
3. 检查 sessionTarget 与 payload.kind 的匹配关系：
   - `isolated` 只能配 `agentTurn`
   - `main` 只能配 `systemEvent`
4. 查看日志确认是否有错误：`openclaw logs`

---

### Q2: Follow-up 任务没有自动触发下一阶段？

**A:** 常见原因：
1. SKILL.md 中未正确定义 follow-up 条件
2. 上一阶段任务执行失败，未满足触发条件
3. 检查任务执行日志，确认上一阶段是否成功完成
4. 确认 follow-up 的触发条件与当前任务输出匹配

---

### Q3: Heartbeat 执行频率不对？

**A:** Heartbeat 默认约 30 分钟执行一次，这是系统设定的。如果需要更频繁的检查，应使用 Cron 任务（支持分钟级精度）。

---

### Q4: 自动化任务执行了但没有输出？

**A:** 排查步骤：
1. 检查 delivery 配置是否正确
2. 确认目标渠道（企业微信/飞书等）连接正常
3. 查看任务执行日志：`openclaw logs`
4. 检查 SKILL.md 中的输出格式定义

---

### Q5: 如何调试正在运行的自动化任务？

**A:** 使用以下命令：
```bash
# 查看所有 Cron 任务
openclaw cron list

# 查看特定任务详情
openclaw cron detail <job-id>

# 手动触发任务进行测试
openclaw cron run <job-id>

# 查看实时日志
openclaw logs --follow
```

---

### Q6: 自动化任务占用资源过高？

**A:** 优化建议：
1. 减少 Heartbeat 检查频率
2. 合并多个 Cron 任务为单个任务
3. 使用 `isolated` session 避免影响主会话
4. 定期清理旧的执行日志

---

### Q7: 如何备份和恢复自动化配置？

**A:** 自动化配置存储在以下位置：
- Cron 配置：`~/.openclaw/openclaw.json`
- 技能文件：`~/.openclaw/workspace/skills/`
- Heartbeat 配置：`~/.openclaw/workspace/HEARTBEAT.md`

定期备份这些文件即可恢复配置。
