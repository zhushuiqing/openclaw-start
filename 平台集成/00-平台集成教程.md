# 📘 第四阶段：OpenClaw 平台集成教程（1 天）

> **培训阶段**：第四阶段 - 平台集成与团队协作
> **目标**：把 OpenClaw 接入企业微信/飞书/钉钉，随时随地使用
> **预计用时**：3-4 小时

---

## 📋 学习目标清单

完成本阶段培训后，学员将能够：

1. **理解平台集成原理**：掌握 Webhook、API 回调等集成机制
2. **接入企业微信**：完成自建应用创建、配置和测试全流程
3. **接入飞书**：完成飞书开放平台应用创建和事件订阅配置
4. **接入钉钉**：完成钉钉企业内部应用创建和机器人配置
5. **接入国际平台**：完成 Telegram 和 Discord 的 Bot 接入
6. **多平台同时管理**：配置和管理多个消息平台的并发请求
7. **排查集成问题**：诊断和解决回调失败、消息丢失等常见问题

## 一、平台集成核心概念

### 1.1 为什么接入消息平台？

| 优势 | 说明 | 示例 |
|------|------|------|
| **便捷性** | 不用打开终端，在聊天界面就能和 AI 对话 | 手机随时使用 |
| **协作性** | 团队成员都能使用 | 共享 AI 助手 |
| **记录保存** | 消息记录自动保存 | 方便回溯和审计 |
| **自动化** | 与 Cron 任务结合实现自动推送 | 每日晨报自动发到群里 |

### 1.2 集成方式对比

| 平台 | 集成方式 | 配置复杂度 | 审核要求 |
|------|---------|-----------|---------|
| 企业微信 | 自建应用 + API 回调 | 中等 | 需要管理员权限 |
| 飞书 | 企业自建应用 + 事件订阅 | 中等 | 需要审核发布 |
| 钉钉 | 企业内部应用 + 机器人 | 简单 | 需要管理员权限 |
| Telegram | Bot Token | 简单 | 无需审核 |
| Discord | Bot Token + OAuth2 | 简单 | 无需审核 |

## 二、接入前准备

### 2.1 系统要求

- [ ] OpenClaw 已安装并正常运行
- [ ] Gateway 正常运行（`openclaw gateway status`）
- [ ] 服务器有公网 IP 或已配置内网穿透
- [ ] 有企业管理员权限（创建应用需要）

### 2.2 网络要求

| 平台 | 端口 | 协议 | 公网要求 |
|------|------|------|---------|
| 企业微信 | 18789 | HTTP/HTTPS | 需要公网 IP |
| 飞书 | 18789 | HTTP/HTTPS | 需要公网 IP |
| 钉钉 | 18789 | HTTP/HTTPS | 需要公网 IP |
| Telegram | - | HTTPS | 需要代理（国内） |
| Discord | - | HTTPS | 需要代理（国内） |

## 三、实战案例

### 实战 1：接入企业微信

#### 步骤 1：创建自建应用

1. 登录企业微信管理后台：`https://work.weixin.qq.com`
2. 点击 **应用管理** → **创建应用**
3. 填写应用名称（如"AI 助手"）
4. 上传应用图标（可选）
5. 点击 **创建**

#### 步骤 2：获取配置信息

| 配置项 | 获取位置 | 示例 |
|--------|---------|------|
| **CorpID** | 我的企业 → 企业信息底部 | `ww_xxxx` |
| **AgentID** | 应用详情页 | `1000001` |
| **Secret** | 应用详情页 → 查看 | `xxxx` |

#### 步骤 3：配置回调

1. 找到 **接收消息** → **设置 API 接收**
2. 填写 URL：`http://你的服务器 IP:18789/webhook/wecom`
3. 填写 Token 和 EncodingAESKey
4. 点击 **保存**

#### 步骤 4：配置 OpenClaw

编辑 `~/.openclaw/openclaw.json`：

```json
{
  "channels": {
    "wecom": {
      "corpId": "ww_xxxx",
      "agentId": "1000001",
      "secret": "xxxx",
      "token": "abc123",
      "encodingAesKey": "xxxx",
      "enabled": true
    }
  }
}
```

#### 步骤 5：重启并测试

```bash
openclaw gateway restart
```

### 实战 2：接入飞书

#### 步骤 1-3：创建应用并获取凭证

1. 打开飞书开放平台：`https://open.feishu.cn`
2. 创建企业自建应用 → 添加机器人能力
3. 获取 App ID 和 App Secret

#### 步骤 4：配置事件订阅

1. 配置请求地址：`http://你的服务器 IP:18789/webhook/feishu`
2. 订阅事件：`im.message.receive_v1`

#### 步骤 5：配置 OpenClaw

```json
{
  "channels": {
    "feishu": {
      "appId": "cli_xxxx",
      "appSecret": "xxxx",
      "enabled": true
    }
  }
}
```

## 四、多平台同时接入

### 4.1 配置示例

```json
{
  "channels": {
    "wecom": { ... },
    "feishu": { ... },
    "dingtalk": { ... },
    "telegram": { ... },
    "discord": { ... }
  }
}
```

### 4.2 平台管理命令

```bash
# 查看所有已接入平台
openclaw channels list

# 启用平台
openclaw channels enable wecom

# 禁用平台
openclaw channels disable telegram

# 测试平台连接
openclaw channels test wecom
```

## 🔍 常见问题排查

### Q1: 企业微信机器人只能发不能收？
**A:** 群机器人只能发送。需要使用**自建应用**才能收发消息。

### Q2: 飞书回调验证失败？
**A:** 确保 Gateway 正在运行，且 URL 可访问。检查防火墙是否开放 18789 端口。

### Q3: Telegram 连接超时？
**A:** 国内网络需要代理：`export HTTPS_PROXY=http://127.0.0.1:7890`

### Q4: 钉钉回调返回 404？
**A:** 确认 Gateway 正常运行，回调地址格式正确：`http://IP:18789/webhook/dingtalk`

### Q5: 多平台消息混乱？
**A:** 检查各平台 channel 配置，确认 enabled 字段正确，必要时重启 Gateway。

### Q6: 如何调试？
**A:** `tail -f ~/.openclaw/gateway/gateway.log`

### Q7: 如何备份配置？
**A:** 备份 `~/.openclaw/openclaw.json`，妥善保管平台凭证。
